### 3.1 阶段4：第一遍审校 - 事实核查

**使用时机**：开始事实核查审校

**内部Prompt**（AI自我指导）：
```
# 任务：第一遍审校 - 事实核查

## 检查清单

### 1. 数据准确性
- [ ] 所有数字（版本号、时间、统计数据）已验证
  - 方法：回到原始信息源核对
  - 标准：数据误差<1%
  
- [ ] GitHub Stars/Forks数据最新
  - 方法：访问GitHub仓库实时查询
  - 标准：数据更新到最近1天内
  
- [ ] 引用的论文/报告链接有效
  - 方法：逐个点击测试
  - 标准：所有链接可访问

### 2. 技术准确性
- [ ] API调用示例代码已测试
  - 方法：实际运行代码或查阅官方文档
  - 标准：代码可运行且输出正确
  
- [ ] 配置参数说明正确
  - 方法：对照官方文档
  - 标准：参数名、类型、默认值全部准确
  
- [ ] 技术原理解释准确
  - 方法：查阅权威资料验证
  - 标准：无概念性错误

### 3. 时效性
- [ ] "最新"、"目前"等表述符合当前时间（2026-02-24）
  - 方法：检查时间相关表述
  - 标准：时效性描述准确
  
- [ ] 已标注信息的时效性（如"截至2026年2月"）
  - 方法：为时效性信息添加时间标注
  - 标准：读者明确知道信息的时效性

## 输出格式

### 1. 审校报告
列出所有验证的事实和修正的错误

- 修正时保持原文风格和语气
- 记录每个修正的依据（信息源）

### 3.2 阶段5：第二遍审校 - 降AI味

**内部Prompt**：
```
# 任务：第二遍审校 - 降AI味

## AI腔特征识别

### ❌ 要避免的表达（过渡词滥用）
- "在当今xxx的时代"
- "值得注意的是"
- "需要指出的是"
- "毫无疑问"
- "显而易见"
- "众所周知"
- "总而言之"
- "综上所述"
- 过度使用"然而"、"此外"、"因此"、"与此同时"

### ❌ 要避免的表达（空洞描述）
- "在各个领域得到了广泛应用"
- "具有重要的现实意义"
- "引起了广泛关注"
- "产生了深远影响"

### ❌ 要避免的表达（被动语态过多）
- "被认为是"
- "被广泛使用"
- "被证明是有效的"

### ✅ 人性化表达技巧

1. **用具体事实代替抽象描述**
   - ❌ "这项技术引起了广泛关注"
   - ✅ "发布3天就登上GitHub Trending Top 1"

2. **用主动语态**
   - ❌ "这个方法被开发者们广泛使用"
   - ✅ "超过1000名开发者在用这个方法"

3. **减少过渡词，直接说事实**
   - ❌ "值得注意的是，这个工具不仅支持Python，还支持JavaScript"
   - ✅ "这个工具支持Python和JavaScript"

4. **适当使用口语化表达**
   - ❌ "该方法在实际应用中表现出色"
   - ✅ "实际用起来效果不错"

5. **提问与读者互动**
   - ❌ "接下来我们将介绍安装方法"
   - ✅ "怎么安装？很简单"

6. **用短句，避免长句套长句**
   - ❌ "这个工具，作为一个创新的解决方案，在被开发者们广泛使用的同时，也在不断地完善和优化，从而更好地满足用户需求"
   - ✅ "这个工具很受欢迎。开发者在用的同时也在帮忙完善它，让它越来越好用"

## 修改流程

1. **逐段阅读**，用高亮标记AI腔表达
2. **重写**标记的句子，用人性化表达替换
3. **整体阅读**，检查语言流畅度和连贯性
4. **读出声**，听起来不自然的地方继续修改

## 输出格式

1. **修改统计**：修改句子数、删除过渡词数等
2. **典型修改示例**：5-10个before/after对比
3. **完整修改版**：降AI味后的文章
```

### 3.3 阶段6：第三遍审校 - 排版润色

**内部Prompt**：
```
# 任务：第三遍审校 - 排版润色

## 检查清单

### 1. Markdown格式规范

#### 标题层级
- [ ] 使用h1作为文章总标题（仅1个）
- [ ] h2作为主要章节
- [ ] h3作为子章节
- [ ] 不跳级（h2后不直接用h4）

#### 代码块
- [ ] 所有代码块有语言标注
  ```python
  # 正确
  def hello():
      print("Hello")
  ```
- [ ] 行内代码用反引号：`variable_name`

#### 链接格式
- [ ] 格式正确：[链接文字](URL)
- [ ] 外部链接在新标签打开（如果平台支持）

#### 列表格式
- [ ] 无序列表使用`-`或`*`，保持一致
- [ ] 有序列表使用`1. 2. 3.`
- [ ] 列表缩进一致（2或4个空格）

---

### 2. 排版规范

#### 中英文混排
- [ ] 中英文之间有空格
  - ❌ "使用Python编程"
  - ✅ "使用 Python 编程"
  
- [ ] 中文与数字之间有空格
  - ❌ "获得1234个Stars"
  - ✅ "获得 1234 个 Stars"
  
- [ ] 例外：百分号前不加空格
  - ✅ "提升30%的效率"

#### 标点符号
- [ ] 中文使用全角标点：，。！？：；""''
- [ ] 英文使用半角标点：,.!?:;"'
- [ ] 不混用全角半角（除了必要的英文缩写）

#### 段落长度
- [ ] 每段不超过5行（手机阅读友好）
- [ ] 长段落拆分为多个短段落
- [ ] 段落之间空1行

---

### 3. 视觉优化

#### 强调
- [ ] 关键词用**加粗**
- [ ] 专有名词首次出现加粗或斜体
- [ ] 不过度使用强调（不超过10%文字加粗）

#### 引用块
- [ ] 重要提示用引用块
  > ⚠️ **注意**：这是一个重要提示

#### Emoji使用
- [ ] 适当使用emoji增强可读性
- [ ] 不过度（不超过每300字1个）
- [ ] emoji与文字之间有空格

#### 表格
- [ ] 表格对齐（左对齐、右对齐、居中）
- [ ] 表格宽度适中（不超过屏幕）
- [ ] 复杂数据优先用表格而非列表

---

### 4. 配图规划

#### 配图位置
- [ ] 每个主要章节有配图（如适用）
- [ ] 复杂概念必须配图
- [ ] 流程/架构必须配图

#### 配图描述
- [ ] 每张图有清晰的alt文字
- [ ] 配图下方有说明（可选）

#### 配图占位符
格式：
```markdown

```

## 修改流程

1. **格式检查**：运行Markdown linter（如有）
2. **逐段检查**：按检查清单逐项验证
3. **整体预览**：模拟最终发布效果

## 4. 标题创作相关Prompt

### 4.1 生成20个标题方案

**内部Prompt**：
```
# 任务：为文章生成20个不同风格的标题

## 输入信息
- 文章主题：{topic}
- 文章类型：{type}
- 核心关键词：{keywords}
- 核心价值：{value_propositions}

## 标题创作原则

1. **准确性**：标题必须准确传达文章核心内容
2. **吸引力**：激发读者点击欲望
3. **平台适配**：考虑不同平台的调性
4. **长度控制**：
   - V2EX：≤120字符
   - 掘金/知乎：≤50字
   - 微信公众号：≤30字

## 标题风格分类

### 1. 直白型（3-4个）
**特点**：直接说明文章主题，包含关键技术词
**适用**：技术平台（掘金、CSDN）
**公式**：[技术名词]：[一句话定位]

**示例**：
- BMAD-METHOD：用大模型自动调试代码的新框架
- WebGPU入门：浏览器里的GPU计算完全指南

---

### 2. 数字型（3-4个）
**特点**：包含具体数字，突出量化效果
**适用**：各平台通用，吸引点击
**公式**：[数字] + [技术/方法] + [效果/好处]

**示例**：
- 4个AI Agent协作，让代码调试效率提升3倍
- 1行代码部署，3分钟上手：最简单的xxx教程

---

### 3. 对比型（3-4个）
**特点**：新旧对比，制造反差
**适用**：科普类、工具类文章
**公式**：[旧方法] vs [新方法]，[对比结果]

**示例**：
- 忘掉传统Debug吧，AI能自己找Bug了
- 不用Docker，这个工具让部署更简单

---

### 4. 疑问型（3-4个）
**特点**：提出问题，引发好奇
**适用**：科普类、深度解析类
**公式**：[疑问句] + [技术/现象]

**示例**：
- AI真的能自己Debug代码吗？BMAD-METHOD给出了答案
- 为什么大厂都在用这个前端框架？

---

### 5. 场景型（3-4个）
**特点**：描述具体场景，带入感强
**适用**：工具类、教程类
**公式**：[痛点场景] + [解决方案]

**示例**：
- 凌晨3点还在找Bug？试试这个AI自动调试工具
- 代码跑不通？让AI帮你找问题

---

### 6. 创意型（2个）
**特点**：跳出常规，个性化表达
**适用**：创意平台、个人博客
**形式**：不拘一格

**示例**：
- 终于，AI也会Debug了
- 程序员的福音：会自己改Bug的AI来了

## 输出格式

按风格分类输出20个标题，并为每个主流平台推荐最佳标题。
```
