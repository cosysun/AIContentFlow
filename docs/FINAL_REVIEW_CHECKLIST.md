# 📋 最终审阅规则 - Final Review Checklist

> **版本**: v1.0  
> **更新日期**: 2026-03-05  
> **适用阶段**: 八段式写作工作流 - 阶段6（最终审阅）

---

## 概述

最终审阅是内容发布前的最后一道质量关卡，**不仅仅是格式检查**，必须对以下三个维度进行系统性校验：

1. **数据真实性校验** - 确保每条数据可溯源、有时效标注
2. **可操作性验证** - 确保读者能真正跟着做
3. **文章质量评分** - 量化输出，决定是否可发布

---

## A. 数据真实性校验

### 校验对象
文中所有具体数字和数据，包括但不限于：
- 模型参数量（如"7B参数"、"1.8T tokens"）
- 性能指标（如"准确率提升30%"、"MMLU得分92.1"）
- 市场数据（如"用户量突破1亿"、"融资金额XX亿"）
- 时间节点（如"2024年Q3发布"）
- 比较数据（如"比GPT-4快3倍"）

### 校验标准

| 数据类型 | 要求 |
|---------|------|
| 有明确来源 | 标注来源（论文/官方公告/权威媒体） |
| 无法溯源 | 必须**删除**或标注 `[待验证]` |
| 超过6个月 | 必须标注 `（截至XXXX年XX月）` |
| 预测/估算数据 | 必须标注 `（预测值）` 或 `（估算）` |

### 执行步骤
1. 扫描全文，列出所有具体数字
2. 逐一核对来源，能找到原始出处的保留并标注
3. 无法核实的数据，给出处理方案（删除/替换/标注）
4. 输出"数据溯源清单"

---

## B. 可操作性验证

### 校验对象
- 操作步骤（如"第一步：打开设置..."）
- 工具推荐（如"使用XX工具可以..."）
- 代码示例
- 外部链接
- 配置参数

### 校验标准

| 问题类型 | 处理方式 |
|---------|---------|
| 步骤不清晰 | 补充截图说明或更详细描述 |
| 工具已下线/改版 | 更新为最新版本操作，或提供替代方案 |
| 代码无法运行 | 修复代码，标注测试环境 |
| 链接失效 | 更新链接或删除 |
| 前置条件未说明 | 在步骤前补充"前置条件"说明 |

### 执行步骤
1. 列出文中所有操作指引
2. 逐一验证可行性（重点检查软件版本、API变更）
3. 标出"潜在障碍点"并给出解决方案
4. 确认文章整体逻辑流畅，读者能从头跟到尾

---

## C. 文章质量评分卡

最终审阅必须输出以下格式的评分卡：

```
┌─────────────────────────────────────────────────────────┐
│  📊 文章质量评分卡                                        │
│                                                          │
│  信息密度：   ★★★★☆  (4/5)  信息量充足，重点突出          │
│  数据支撑：   ★★★☆☆  (3/5)  部分数据需补充来源            │
│  可读性：     ★★★★★  (5/5)  语言流畅，结构清晰            │
│  可操作性：   ★★★★☆  (4/5)  步骤清晰，可跟随执行          │
│  AI味指数：   ★☆☆☆☆  (1/5)  自然度高（越低越好）          │
│                                                          │
│  综合得分：   XX/25                                       │
│  发布建议：   ✅ 可发布 / ⚠️ 需修改后发布 / ❌ 建议重写    │
│                                                          │
│  修改建议：                                               │
│  1. [具体修改点1]                                         │
│  2. [具体修改点2]                                         │
└─────────────────────────────────────────────────────────┘
```

### 评分标准

#### 信息密度 (1-5)
- 5分：每段都有实质性信息，无废话
- 3分：有少量填充内容，但整体充实
- 1分：大量套话，信息稀少

#### 数据支撑 (1-5)
- 5分：所有关键论点都有数据/案例支撑，来源清晰
- 3分：部分论点有支撑，部分靠推断
- 1分：几乎全是主观判断，无数据

#### 可读性 (1-5)
- 5分：语言自然流畅，结构清晰，读者体验极佳
- 3分：基本通顺，但有些地方略显生硬
- 1分：逻辑混乱，语言机械

#### 可操作性 (1-5)
- 5分：所有步骤清晰可执行，读者能立即上手
- 3分：主要步骤清晰，部分细节需要读者自行摸索
- 1分：只有概念，无具体操作指引

#### AI味指数 (1-5，越低越好)
- 1分：完全看不出AI生成痕迹
- 3分：偶尔有套路化表达
- 5分：明显AI腔，机械感强

### 发布建议阈值

| 综合得分 | AI味指数 | 建议 |
|---------|---------|------|
| ≥20分 | ≤2分 | ✅ 可直接发布 |
| 15-19分 | ≤3分 | ⚠️ 需修改后发布（列出具体修改点） |
| <15分 或 AI味≥4分 | - | ❌ 建议重写 |
