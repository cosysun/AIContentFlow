# AI助手是怎么工作的？一篇文章讲透Agent、MCP、Rules、Skills

> **写在前面**：这是一篇给所有人看的AI科普文章。如果你听说过ChatGPT，想知道它背后的工作原理，或者好奇"AI助手"到底是怎么帮你完成任务的——这篇文章就是为你准备的。我们会用最通俗的语言，配合生活中的例子，带你理解AI时代的四个核心概念。

---

## 一、AI助手的进化史：从"对话机器"到"真正的助手"

### 1.1 三个发展阶段

想象一下你雇佣助手的过程：

**阶段1：只会回答问题的客服（2022-2023）**

```
你：今天天气怎么样？
ChatGPT：根据我的训练数据...（但其实它不知道实时天气）
```

这个阶段的AI就像一个只会背书的学生——知识丰富，但不会查资料，也不能帮你做事。

---

**阶段2：会用工具的实习生（2023-2024）**

```
你：帮我查一下明天北京的天气
AI：好的，让我调用天气API...（调用工具）
AI：明天北京晴天，最高温度25°C
```

进步在哪？AI学会了"使用工具"！就像实习生学会了用电脑查资料、用Excel做表格。

---

**阶段3：能自主规划的正式员工（2024-现在）**

```
你：帮我准备下周的旅行
AI：好的，我来规划一下：
    1. 先查目的地天气（调用天气工具）
    2. 再搜索机票酒店（调用旅游网站API）
    3. 制定行程计划（调用地图工具）
    4. 整理成一份文档（调用文档工具）
    （自动完成，10分钟后通知你）
```

这就是今天的**AI Agent**（智能体）——不仅会用工具，还会自己规划任务步骤！

---

### 1.2 什么让AI从"客服"变成了"助手"？

答案是四样东西：

| 组件 | 角色 | 生活类比 |
|------|------|---------|
| **Agent** | 大脑 | 你雇佣的助理本人 |
| **MCP** | 工具箱 | 助理使用的各种工具（电脑、电话、车） |
| **Rules** | 工作手册 | 公司的规章制度 |
| **Skills** | 专业技能 | 助理学过的专业培训（财务、法律、设计） |

接下来，我们一个个讲解。

## 二、Agent（智能体）：AI的"大脑"

### 2.1 什么是Agent？

**Agent = AI + 思考能力 + 行动能力**

想象你雇了一个助理：
- **普通员工**：你说一句，TA做一件事
- **优秀助理（Agent）**：你说"准备年会"，TA自己分解成50个任务并逐一完成

Agent的核心能力：
1. **理解**你的意图
2. **规划**任务步骤
3. **执行**每个步骤
4. **记住**过程中的信息
5. **主动**向你汇报进度

---

### 2.2 Agent是怎么思考的？

#### 方法1：ReAct思维链（边想边做）

就像你解数学题时的草稿纸：

```
你的任务：分析最近的科技热点

Agent的思考过程：
┌─────────────────────────────┐
│ 想法1: 我需要找最新的科技新闻 │
│ 行动1: 搜索"科技新闻 2024"   │
│ 观察1: 找到50篇文章           │
├─────────────────────────────┤
│ 想法2: 太多了，筛选热度最高的│
│ 行动2: 分析每篇的阅读量       │
│ 观察2: 前10篇明显更热门       │
├─────────────────────────────┤
│ 想法3: 现在整理成报告吧       │
│ 行动3: 生成Markdown文档      │
│ 完成！                       │
└─────────────────────────────┘
```

**关键**：每做一步，Agent都会"停下来想想"，这样不容易跑偏。

---

#### 方法2：Plan-and-Execute（先计划再执行）

就像旅行前制定详细行程：

```
你的任务：写一篇产品分析文章

Agent的计划：
第1步：研究产品功能（预计10分钟）
第2步：对比竞品（预计20分钟）
第3步：撰写文章主体（预计30分钟）
第4步：配图和排版（预计15分钟）

然后按顺序执行...
```

**优势**：不会漏掉步骤，时间可控。

---

### 2.3 Agent的"记忆"

就像人的记忆分短期和长期：

| 记忆类型 | 作用 | 例子 |
|---------|------|------|
| **短期记忆** | 记住当前对话 | "你刚才说要去北京对吧？" |
| **长期记忆** | 记住你的习惯 | "你上次说不喜欢吃香菜" |
| **情景记忆** | 记住过去的任务 | "上次帮你订机票用的是携程" |

这样AI才能越用越懂你！

## 三、MCP（模型上下文协议）：AI的"工具箱"

### 3.1 什么是MCP？

MCP = **让AI连接各种工具的"万能转接口"**

**生活类比**：

以前你买电器：
- 买美国货 → 需要美标插头
- 买日本货 → 需要日标插头
- 买欧洲货 → 需要欧标插头

有了**万能转换插头**后：
- 不管买哪国的，插上就能用！

MCP就是AI世界的"万能转换插头"。

---

### 3.2 MCP解决了什么问题？

**以前**：每个AI要单独对接每个工具

```
Claude要对接：
- GitHub ✓（自己写代码）
- Gmail ✓（又写一遍代码）
- 数据库 ✓（再写一遍代码）

ChatGPT要对接：
- GitHub ✓（重复写代码）
- Gmail ✓（重复写代码）
- 数据库 ✓（重复写代码）
```

太浪费时间了！

---

**现在**：有了MCP标准接口

```
工具方写一次 MCP接口 → 所有AI都能用
- GitHub发布 GitHub MCP → 所有AI都能管理代码
- Google发布 Gmail MCP → 所有AI都能发邮件
- MySQL发布 Database MCP → 所有AI都能查数据
```

就像USB接口：一个接口，所有设备通用。

---

### 3.3 MCP怎么工作？

三个角色：

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│ AI应用    │ ←──→ │ MCP协议  │ ←──→ │ 工具服务  │
│ (Claude) │      │ (翻译官) │      │ (GitHub) │
└──────────┘      └──────────┘      └──────────┘

对话过程：
1. AI: "我想搜索代码里的Bug"
2. MCP协议: "翻译成GitHub能理解的命令"
3. GitHub: "找到了23个可能的Bug"
4. MCP协议: "翻译成AI能理解的格式"
5. AI: "好的，我整理成报告告诉用户"
```

**你只需要知道**：MCP让AI能用上成千上万种工具，不需要每次都重新开发。

---

### 3.4 MCP的实际应用

#### 例子1：代码助手

```
你：帮我分析这个项目的代码质量

Agent通过MCP调用：
- GitHub MCP → 读取代码文件
- ESLint MCP → 检查代码规范
- Pytest MCP → 运行测试
- Markdown MCP → 生成报告

10分钟后给你一份完整的代码质量报告
```

#### 例子2：数据分析

```
你：分析一下上个月的销售数据

Agent通过MCP调用：
- Database MCP → 从MySQL提取数据
- Pandas MCP → 数据清洗和计算
- Matplotlib MCP → 生成图表
- Email MCP → 发送报告到你邮箱
```

## 四、Rules（规则）：AI的"工作手册"

### 4.1 为什么需要Rules？

AI就像一个极其聪明但没有常识的实习生：

**没有Rules的后果**：

```
你：帮我写个Python脚本
AI：好的！（写了500行代码，没有任何注释，变量名是a、b、c）

你：帮我写个邮件
AI：尊敬的先生/女士，本人谨代表敝司...（太正式了）

你：帮我设计个Logo
AI：（生成了一个丑到爆炸的图）
```

**有了Rules后**：

```
Rule 1: Python代码必须有类型注解和文档注释
Rule 2: 邮件语气轻松友好，不要太正式
Rule 3: Logo设计必须符合品牌色系（蓝色#1E90FF）
```

现在AI输出的质量就有保证了！

---

### 4.2 Rules的四个优先级

就像公司的规章制度：

```
┌────────────────────────────────┐
│  🔴 P0：必须遵守（违反直接拒绝）│
│  例：绝不能泄露用户隐私           │
│  例：代码必须通过安全检查         │
└────────────────────────────────┘
         ↓ 覆盖
┌────────────────────────────────┐
│  🟡 P1：强烈建议（违反警告提示）│
│  例：函数不应超过50行             │
│  例：必须写单元测试               │
└────────────────────────────────┘
         ↓ 覆盖
┌────────────────────────────────┐
│  🟢 P2：最佳实践（优化建议）    │
│  例：使用列表推导而不是for循环   │
│  例：变量名应该有意义             │
└────────────────────────────────┘
         ↓ 覆盖
┌────────────────────────────────┐
│  ⚪ 默认：AI自己的习惯           │
│  （训练数据中学到的习惯）         │
└────────────────────────────────┘
```

---

### 4.3 好的Rules长什么样？

#### ❌ 坏的Rule（模糊）

```
Rule: 代码要写得好
```

AI：什么叫"好"？（困惑）

---

#### ✅ 好的Rule（具体）

```
Rule: Python函数必须满足：
1. 有类型注解（输入和输出）
2. 有文档注释（说明功能、参数、返回值）
3. 长度不超过50行
4. 不使用eval()或exec()

好例子：
def calculate_total(prices: list[float]) -> float:
    """计算总价
    
    Args:
        prices: 价格列表
    Returns:
        总价（保留2位小数）
    """
    return round(sum(prices), 2)

坏例子：
def calc(p):  # ❌ 没有类型注解和文档
    return sum(p)
```

**关键**：要有正反例子，AI才知道怎么做。

---

### 4.4 Rules的实际应用

#### 场景1：内容创作

```
Rules:
- 标题必须有emoji和数字（"🚀 10个技巧"）
- 正文段落不超过4行
- 每3段插入一个列表或表格
- 禁止使用"赋能""闭环"等AI味词汇
- 多用生活化类比，少用专业术语
```

结果：产出的文章更容易阅读！

#### 场景2：代码审查

```
Rules:
- 安全：禁止使用eval()、exec()、pickle.loads()
- 性能：数据库查询必须有索引
- 可维护性：复杂逻辑必须有注释
- 测试：核心函数必须有单元测试
```

结果：代码质量有保障！

## 五、Skills（技能）：AI的"专业培训"

### 5.1 什么是Skills？

**Skill = 针对特定任务的"工作流程模板"**

**生活类比**：

你雇的助理虽然聪明，但让TA：
- 做财务报表 → 不会（需要**财务Skill**）
- 设计PPT → 不会（需要**设计Skill**）
- 写法律合同 → 不会（需要**法律Skill**）

**解决方案**：给TA培训，或者准备一套"操作手册"。

Skill就是这套**操作手册**。

---

### 5.2 Skill长什么样？

#### 例子：竞品分析Skill

```
技能名称：competitor-analysis

工作流程：
第1步：明确分析目标
  - 询问：要分析哪个产品？关注哪些方面？
  - 输出：分析计划

第2步：信息收集（15分钟）
  - 使用MCP工具：
    ✓ 搜索引擎 → 找产品官网
    ✓ 社交媒体API → 找用户评价
    ✓ App商店API → 找下载和评分数据
  - 输出：原始数据

第3步：数据整理（10分钟）
  - 按功能、价格、用户评价分类
  - 应用Rules：数据必须可视化（图表）
  - 输出：结构化数据

第4步：撰写报告（20分钟）
  - 使用模板：
    * 产品概况（2段）
    * 核心功能对比（表格）
    * 优劣势分析（列表）
    * 建议（3-5条）
  - 应用Rules：语言简洁，有理有据
  - 输出：Markdown文档

第5步：生成摘要
  - 提炼3条核心结论
  - 输出：一段话摘要

预计总时长：45分钟
成功率：95%
```

---

### 5.3 Skills的价值

#### 没有Skill：

```
你：帮我分析一下竞品
Agent：好的...(一头雾水，不知道从哪开始)
      我搜索了一下...(结果混乱)

```

#### 有了Skill：

```
你：帮我分析一下竞品
Agent：已加载【竞品分析Skill】
      第1步：明确目标 ✓
      第2步：收集数据 ✓（找到23篇相关文章）
      第3步：整理数据 ✓（生成3个对比表格）
      第4步：撰写报告 ✓（2500字，5个图表）
      第5步：生成摘要 ✓
      
      完成！这是报告链接...
```

**区别**：有Skill就是有专业培训，质量稳定，效率更高。

## 七、实战案例：自动化内容创作系统

让我们看一个真实的应用场景。

### 7.1 需求

**目标**：每天自动监控科技热点，选择热门话题，自动创作深度文章

**挑战**：
- 要找最新的热点（需要联网搜索）
- 要写出高质量文章（6000+字）
- 要符合发布规范（格式、风格）
- 要完全自动化（不需要人工干预）

---

### 7.2 解决方案

#### 系统架构

```
定时触发器
    ↓
【监控阶段】
    ↓
Agent加载 → hot-topic-monitor Skill（热点监控技能）
    ↓
通过MCP调用3个工具：
- brave_search MCP → 搜索HackerNews
- github MCP → 搜索GitHub Trending
- web_search MCP → 搜索TechCrunch
    ↓
分析热度，生成Top 5选题
    ↓
【通知用户】
用户选择选题2
    ↓
【创作阶段】
    ↓
Agent加载 → eight-stage-writing Skill（八段式写作技能）
    ↓
阶段1：确认选题 ✓
阶段2：深度调研（调用15+个MCP工具） ✓
阶段3：创作内容（6000-8000字） ✓
阶段4：三遍审校（应用Rules：降AI味） ✓
阶段5：拟定标题（20个方案） ✓
阶段6：最终审阅 ✓
阶段7：保存草稿 ✓
阶段8：制定推广计划 ✓
    ↓
【完成】
通知用户："文章已完成，点击查看"
```

---

### 7.3 关键技术

#### Skills部分

```
技能1：hot-topic-monitor（热点监控）
- 定义：如何判断一个话题是否热门
- 流程：搜索→评分→排序→推荐
- 时长：5分钟

技能2：eight-stage-writing（八段式写作）
- 定义：如何创作高质量文章
- 流程：8个阶段，每个阶段有详细步骤
- 时长：60-90分钟
```

#### MCP工具

```
brave_search → 搜索引擎
github → 代码仓库
web_search → 通用网页搜索
read_file → 读取参考文档
write_file → 保存草稿
notify → 通知用户
```

#### Rules规范

```
P0级：必须遵守
- 禁止抄袭原文
- 必须注明信息来源
- 敏感话题需要人工审核

P1级：强烈建议
- 文章长度6000-8000字
- 段落不超过4行
- 每3段插入列表或表格

P2级：最佳实践
- 多用生活化类比
- 避免"赋能""闭环"等AI味词汇
- 标题有emoji和数字
```

---

### 7.4 实际效果

| 指标 | 数据 |

## 八、如何自己搭建一个AI助手？

### 8.1 六步流程

```
第1步：需求分析
  你想让AI帮你做什么？
  - 自动回复邮件？
  - 生成周报？
  - 监控竞品？

第2步：选择工具（MCP）
  需要哪些外部能力？
  - 读写文件？→ 文件系统MCP
  - 发送邮件？→ Gmail MCP
  - 查询数据？→ 数据库MCP

第3步：定义流程（Skills）
  写一份"操作手册"
  - 第1步做什么
  - 第2步做什么
  - ...

第4步：制定规范（Rules）
  输出要符合什么标准？
  - 邮件不能超过200字
  - 必须有礼貌用语
  - ...

第5步：配置Agent
  把工具、流程、规范都告诉AI

第6步：测试运行
  试几次，发现问题就调整
```

### 8.2 入门项目推荐

#### 项目1：自动周报生成器（难度：⭐）

```
功能：每周五下午自动生成本周工作总结

需要的MCP：
- Calendar MCP（读日历）
- Email MCP（读邮件）
- Notion MCP（读工作记录）

Skill：
1. 收集本周的会议记录
2. 收集本周的邮件关键词
3. 收集本周的任务完成情况
4. 整理成周报格式
5. 发送给领导

Rules：
- 周报不超过500字
- 突出关键成果
- 用数据说话（完成X个任务）
```

#### 项目2：智能客服机器人（难度：⭐⭐）

```
功能：自动回答用户常见问题

需要的MCP：
- Knowledgebase MCP（读取FAQ知识库）
- Slack/企微 MCP（接收和发送消息）

Skill：
1. 理解用户问题
2. 搜索知识库
3. 生成友好的回答
4. 如果答不上来，转人工

Rules：
- 回答要简洁（不超过100字）
- 语气友好，不能生硬
- 必须说明信息来源
```

#### 项目3：竞品监控助手（难度：⭐⭐⭐）

```
功能：每天监控竞品动态并生成简报

需要的MCP：
- Web Search MCP（搜索网页）
- Social Media MCP（社交媒体）
- Database MCP（存储历史数据）

Skill：
1. 搜索竞品相关新闻
2. 对比昨天的数据，找变化
3. 分析重要性（用AI判断）
4. 生成简报（突出重点）
5. 发送通知

Rules：
- 只报告重要变化（避免信息过载）
- 简报不超过300字
- 必须标注信息来源和时间
```

### Q3：安全吗？会不会泄露隐私？

**风险与防范**：

| 风险 | 防范措施 |
|------|---------|
| **数据泄露** | ✓ 敏感数据加密<br>✓ 使用私有部署<br>✓ 设置Rules禁止分享隐私 |

- 从低风险任务开始（写周报）

**人类擅长的**（创造性工作）：
- 做关键决策
- 处理复杂人际关系
- 创新和创意
- 承担责任

**最好的模式**：人+AI协作
- AI做80%的基础工作
- 人类做20%的关键决策
- 效率提升10倍！
