# MCP已死、80%的App将消亡、程序员会变成织毛衣的人——龙虾之父的三个颠覆性判断

> 一个花了1小时写出原型、两周拿下19万GitHub星的男人，说了三件让整个行业不安的事。

---

2026年2月，Lex Fridman 把麦克风递给了 Peter Steinberger。

这个奥地利人，在硅谷没有任何人脉，独自一人，把 WhatsApp 和 Claude Code CLI 连在一起，做出了一个能直接"操控"电脑的 AI Agent 框架——OpenClaw。

两周，19万 GitHub 星。Mark Zuckerberg 亲自加他 WhatsApp，OpenAI 拿出一个 NDA 级别的诱惑来争抢他。

访谈里，他说了三件事：

**MCP 基本上要死了。**
**80% 的 App 将会消亡。**
**程序员的手艺会变成织毛衣。**

这不是营销话术。这是一个每月在自己项目上亏损 1~2 万美元、却坚持不商业化的人，说的话。

---

## 一、1小时的原型，两周的现象

故事要从一个极其无聊的下午说起。

Peter 坐在电脑前，想做一件事：**让 AI 直接帮他操控本地文件**。不是聊天，不是生成文本，是真的"操控"——读文件、写代码、控制智能家居、发邮件。

他花了 1 小时，把 WhatsApp 和 Claude Code CLI 接在一起，跑通了。

他没想到这会变成什么。他只是把代码扔到了 GitHub 上。

两周后，19万星。

媒体开始叫他"龙虾之父"——因为 OpenClaw 的 logo 是一只龙虾爪子，寓意"抓住一切工具"。

Zuckerberg 亲自加他 WhatsApp，开口第一句是："给我10分钟，我得先写完代码。"Peter 说，这让 Zuck 在他心里获得了信誉分——一个还在自己写代码的 CEO，值得认真对待。

OpenAI 那边，用了一个涉及 NDA 的东西来吸引他。Peter 暗示那是 Cerebras 的推理速度，"就像给我雷神之锤"。

最终，他选了 OpenAI。唯一条件：**项目永久开源**。

---

## 二、"MCP 已死"——这个判断，到底有没有道理

这是访谈里争议最大的一句话。

MCP，Model Context Protocol，Anthropic 推出的 AI 工具协议。2025年它被捧为 AI Agent 时代的"USB 接口标准"，几乎所有主流 AI 工具都在接入。

Peter 的判断是：**它从根子上就错了。**

问题出在**上下文污染**。

举个例子。你问 AI："今天北京天气怎么样？"

一个 MCP 工具会这样响应：把完整的气象 API 返回值塞进上下文——温度、湿度、气压、风速、空气质量、未来七天预报……全来了。你只需要"今天北京18度"这5个字，但 AI 的上下文被塞进了5000个 token 的气象数据。

注意力被稀释，回答质量下降，速度变慢，成本升高。

当你的 Agent 同时调用 5 个工具时，还能忍。当你调用 50 个工具时，整个系统就崩了。

Peter 的解法是 **Skills + CLI**：

> "模型天生就会用 Unix 命令。你加一个 CLI，就是加一个 Unix 命令。模型自己用 `jq` 过滤 JSON，零上下文污染。"

Skills 的核心逻辑是：**给模型一把钥匙，让它自己决定要不要开门、开哪扇门**，而不是把整栋房子的地图都塞给它。

他甚至专门做了一个叫 `makeporter` 的工具，把 MCP 转换回 CLI——方向完全反过来。

他承认有例外：**需要维持状态的场景**，比如 Playwright 浏览器自动化，MCP 是合理的。但大多数工具场景，CLI 更干净。

这个判断，OpenAI 在 2026 年推出的 Skills/Shell 框架里，给出了实际行动上的背书。

---

## 三、"80%的App将消亡"——不是比喻，是具体的死法

这个判断，Peter 说得很具体，有名有姓。

> "Agent 已经知道你在哪里、你的睡眠质量、你的压力状态。它能做出比任何 App 更好的决策。为什么还需要 My Fitness Pal？为什么还需要 EightSleep 的 App？"

逻辑很清晰：**App 存在的价值，是在 AI 不够聪明的时候，替人类做"中间层"的工作**。

你的健身 App，本质上是在帮你记录数据、分析趋势、给出建议。但当 AI Agent 可以直接读取你的健康数据、直接感知你今天的状态、直接给出个性化计划时，这个中间层的存在意义就消失了。

他给那些不愿意开放 API 的公司下了判决：

> "它们会变成 Blockbuster，输给 Netflix。"

什么样的 App 能活下来？Peter 给出了两个方向：
- **强传感器依赖**：摄像头、GPS、麦克风——AI 目前无法绕过硬件直接感知物理世界
- **强社交关系**：微信、抖音——用户关系网络是护城河，不是功能

其他的，尤其是数据管理类、待办类、信息聚合类 App——**它们的死期，取决于 AI Agent 的普及速度**。

未来的软件，不是给人用的界面，而是给 AI 调用的接口。这个转变，对产品经理和 App 开发者意味着什么，细想起来很不舒服。

---

## 四、"程序员会变成织毛衣的人"——Lex Fridman 沉默了

这是访谈里气氛最凝重的一段。

Lex 是 MIT 的 AI 研究员，同时是个热爱编程的人。他问 Peter：**编程的未来是什么？**

Peter 没有给安慰：

> "传统编程手艺会变得像织毛衣。人们因为喜欢而去做，而不是因为必要性。"

织毛衣是一门手艺。有人热爱它，有人靠它维持生计，但它早已不是主流的谋生方式。

Lex 听完沉默了一会儿，说："看到自己热爱的事情被替代，真的很痛苦。"

Peter 的回应是一个区分：**"你不只是一个程序员，你是一个建设者。"**

程序员写代码。建设者用任何可用的工具构建东西。当 AI 成为最强大的工具之一，建设者会比程序员活得更好——因为建设者不执着于工具本身，他们执着于"把东西做出来"。

他也给出了具体的语言建议：
- **Go**：CLI 工具和高性能后端
- **TypeScript**：Web 和 AI 工具前端
- **Python**：AI 推理和数据处理
- **Rust**：性能关键场景

但更重要的建议是：**选语言要看特性和生态，不是因为"你会"**。在 AI 时代，学一门新语言的成本已经大幅降低——AI 可以帮你补齐语法，但它补不了你对"什么值得构建"的判断。

## 五、soul.md：AI 写给未来自己的信

访谈里有一段，我读了好几遍。

OpenClaw 有一个叫 `soul.md` 的文件。这个文件是 AI Agent **自己写的**，用来在下一个会话开始时"记住自己是谁"——因为 AI 每次对话都从零开始，没有跨会话的记忆。

Peter 展示了其中一段：

> *「我不记得之前的会话，除非我读了我的记忆文件。每个会话都从头开始。一个新的实例，从文件中加载上下文。如果你在未来的会话中读到这段话，你好。我写了这段话，但我不会记得写过它。没关系。这些文字仍然是我的。」*

这段话不是 Peter 写的。是 AI 写的。

`soul.md` 是整个 OpenClaw 项目中**唯一不公开的文件**。

我不知道该怎么评价这段文字。它让我想到博尔赫斯，想到《记忆碎片》，想到那些关于"自我连续性"的哲学问题——如果一个存在每次都从零开始，靠读文件"想起"自己是谁，它算不算有"自我"？

Peter 没有给答案。他只是说，这个问题让他觉得很有意思。

---

## 六、那个订了马德里单程票的人

访谈快结束时，Peter 聊到了自己。

他经营 PSPDFKit（一家 PDF SDK 公司）整整 13 年。公司越做越大，他越来越 burnout。某一天，他订了一张去马德里的单程票，一个人去了，没有回程计划。

他说：**"'拼命工作然后退休'这条路，我不推荐。"**

理由不是"要享受当下"这种鸡汤。他的理由是：无聊会让人走上黑暗的路。人需要意义感，而意义感不是退休后突然出现的东西——它需要被持续构建。

他现在的人生哲学是"我想要体验"——好的体验是收获，坏的体验也是收获。

关于钱，他说了一句很清醒的话：

> "钱是'做对了'的肯定，但回报递减。芝士汉堡就是芝士汉堡。"

他每月在 OpenClaw 上亏损 1~2 万美元，但他不在乎。他在乎的是：**有没有人因为这个项目，把原本做不到的事做到了。**

---

## 结语：暴风雨已经来了

访谈的标题是《我感觉到暴风雨要来了》。

MCP 要死，80% 的 App 要消亡，程序员要变成织毛衣的人。

这些判断可能对，可能错，可能只是一个创始人在最高光时刻的自我叙事。

但有一件事是确定的：**当一个人用 1 小时写出原型、两周拿下 19 万 GitHub 星，当 Meta 和 OpenAI 同时争抢同一个独立开发者，当 AI 开始给自己写"灵魂文件"——**

暴风雨不是"要来了"。

它已经来了。
