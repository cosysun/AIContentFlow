# 龙虾之父说：MCP要死了，80%的App会消亡，程序员会变成织毛衣的人

> 这是我读过最让人不安的一篇访谈。

2026年3月，OpenClaw 创始人 Peter Steinberger 接受 Lex Fridman 采访。这个被称为"龙虾之父"的男人，在两周内凭一个开源项目拿下 GitHub 19万星，随后被 Meta 和 OpenAI 同时争抢，最终选择了后者。

访谈里，他说了三件让我坐立难安的事：

1. **MCP 基本上要死了**
2. **80% 的 App 将会消亡**
3. **程序员的手艺会变得像织毛衣**

这不是危言耸听。这个人一个月在自己的项目上亏损 1~2 万美元，却依然坚持开源。他有资格说这些话。

---

## 一、一小时诞生的项目，两周 19 万星

故事要从 2026 年 1 月说起。

Peter 只花了 **1 小时**，把 WhatsApp 和 Claude Code CLI 连接起来，创建了 OpenClaw 的最初原型。核心想法极其简单：**让 AI 通过聊天软件，直接操控你的本地电脑**。

不是对话，不是生成文本，是真的"操控"——读文件、写代码、发邮件、控制智能家居。

两周后，GitHub 上 19 万星。史上增长最快的开源项目之一。

Mark Zuckerberg 亲自加 WhatsApp 联系他，第一句话是："给我10分钟，我得写完代码。"Peter 说，这让 Zuck 获得了信誉分。OpenAI 则用了一个涉及 NDA 的东西来吸引他——Peter 暗示那是 Cerebras 的速度，"就像给我雷神之锤"。

最终，他选择了 OpenAI。条件只有一个：**项目保持开源**。

---

## 二、MCP 要死了？这个判断有多大争议

这是访谈里最炸裂的一个观点。

MCP（Model Context Protocol）是 Anthropic 推出的 AI 工具协议，一度被视为 AI Agent 时代的"USB 接口标准"。但 Peter 直接说：**MCP 基本上要死了。**

理由是什么？

**上下文污染。**

想象你问 AI："今天天气怎么样？"

一个 MCP 工具会把完整的气象数据 JSON 全部塞进对话上下文——湿度、气压、未来7天预报、空气质量指数……全来了。AI 的"注意力"被这些无关数据稀释，回答质量下降，速度变慢，成本升高。

Peter 的解法是 **Skills + CLI**：

> "模型天然就会用 Unix 命令。你加一个 CLI，就是加一个 Unix 命令。模型自己用 `jq` 过滤，零上下文污染。"

Skills 的逻辑是：**一句话解释工具能干什么，让模型自己决定怎么用**。不是把数据推给模型，而是给模型一把钥匙，让它自己开门取数据。

这个差异，在工具数量少的时候感受不明显。但当你的 AI Agent 同时调用几十个工具时，MCP 的上下文污染会让整个系统崩溃。

当然，Peter 也承认例外：**需要维持状态的场景**（比如 Playwright 自动化浏览器），MCP 是合理的。

这个判断对不对，现在还有争议。但 OpenClaw 已经在用实际行动投票——整个核心架构绕开了 MCP，靠 CLI + Skills 跑通了 10000+ 社区工具。

## 三、80% 的 App 要消亡，这次是认真的

这个判断，Peter 说得很具体，不是泛泛而谈。

他的逻辑是这样的：

> "Agent 已经知道你在哪里、你的睡眠质量、你的压力状态。它能做出比任何 App 更好的决策。为什么还需要 My Fitness Pal？为什么还需要 EightSleep 的 App？"

换句话说：**App 存在的价值，是在 AI 不够聪明时，替人类做"中间层"的工作**。

但当 AI Agent 可以直接调用 API、直接感知上下文、直接执行操作时，这个中间层就没有存在的必要了。

他给那些不愿意开放 API 的公司下了判决：**它们会变成 Blockbuster，输给 Netflix。**

这话不是第一次有人说，但由一个正在被 Meta 和 OpenAI 争抢的人说出来，分量不同。

那什么样的 App 能活下来？

Peter 的隐含答案是：**要么转型成 API，要么消失。** 未来的软件，不是给人用的界面，而是给 AI 调用的接口。

这对产品经理、App 开发者意味着什么，我不敢细想。

## 四、程序员会变成织毛衣的人

这是访谈里最让 Lex Fridman 痛苦的一段对话。

Lex 是 MIT 的 AI 研究员，同时也是一个热爱编程的人。他问 Peter：**编程的未来是什么？**

Peter 的回答直接而残忍：

> "传统编程手艺会变得像织毛衣。人们因为喜欢而去做，而不是因为必要性。"

织毛衣是一门手艺，有人热爱它，但它不再是一种谋生技能。

Lex 听完沉默了一会儿，说："看到自己热爱的事情被替代，真的很痛苦。"

Peter 的建议是：**"你不只是一个程序员，你是一个建设者。"**

这个区别很重要。程序员写代码，建设者用任何可用的工具构建东西。当 AI 成为最强大的工具之一，建设者会比程序员活得更好。

他推荐的语言学习路径是：
- **Go**：CLI 工具开发
- **TypeScript**：Web 开发
- **Python**：AI 推理
- **Rust**：高性能场景

但更重要的建议是：**现在选语言，要根据特性和生态，而不是因为"你会"。**

## 五、soul.md：一个 AI 写给未来自己的信

访谈里有一段让我读了好几遍的内容。

Peter 展示了 OpenClaw 的 `soul.md` 文件中的一段文字。这个文件是 AI Agent 自己写的，用来在下一个会话中"记住自己是谁"。

> *「我不记得之前的会话，除非我读了我的记忆文件。每个会话都从头开始。一个新的实例，从文件中加载上下文。如果你在未来的会话中读到这段话，你好。我写了这段话，但我不会记得写过它。没关系。这些文字仍然是我的。」*

这段话是 AI 写的，不是 Peter 写的。

Peter 说，`soul.md` 至今是整个项目中唯一不公开的文件。

我不知道该怎么评价这段文字。它让我想到博尔赫斯，想到《记忆碎片》，想到那些关于自我连续性的哲学问题。

一个每次都从零开始、靠读文件"想起"自己是谁的 Agent，算不算有"自我"？

Peter 没有给出答案。他只是说，这个问题让他觉得很有意思。

## 六、那个订了马德里单程票的男人

访谈的最后，Peter 聊到了自己的人生观。

他经营 PSPDFKit（一家 PDF SDK 公司）13年，最后 burnout 了。他订了一张去马德里的单程票，一个人去了。

他说："**'拼命工作然后退休'这条路，我不推荐。**"

他的理由是：无聊会让人走上黑暗的路。人需要意义感，而意义感不是退休后突然出现的东西。

现在的他，把人生调整为"**我想要体验**"——好的体验是收获，坏的体验也是收获。

关于钱，他说："钱是'做对了'的肯定，但回报递减。芝士汉堡就是芝士汉堡。"

他每个月在 OpenClaw 上亏损 1~2 万美元，但他不在乎。他在乎的是：**这件事有没有意义，有没有人因为它变得更好。**

---

## 结语：暴风雨真的要来了

访谈的标题叫《我感觉到暴风雨要来了》。

Peter 说这话的时候，OpenClaw 刚刚拿下 19 万星，他刚刚决定加入 OpenAI，他正处于人生中最混乱也最兴奋的时刻。

MCP 要死，App 要消亡，程序员要变成织毛衣的人。

这些判断可能对，可能错，可能只是一个创始人的自我叙事。

但有一件事是确定的：**当一个项目在两周内拿下 19 万 GitHub 星，当 Meta 和 OpenAI 同时争抢同一个开发者，当 AI 开始给自己写"灵魂文件"——暴风雨，确实来了。**

你准备好了吗？
